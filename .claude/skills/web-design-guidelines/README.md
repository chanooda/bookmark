# Web Design Guideline

웹 인터페이스 모범 사례 준수를 위해 UI 코드를 검토합니다. 접근성, 성능, UX를 아우르는 100개 이상의 규칙으로 코드를 감사합니다.

## 사용 시점

“내 UI 검토하기”
“접근성 확인하기”
“디자인 감사하기”
“UX 검토하기”
“모범 사례에 따라 내 사이트 확인하기”

## 포함된 범주

- 접근성 (aria-labels, 의미론적 HTML, 키보드 핸들러)
- 포커스 상태 (가시적 포커스, 포커스-가시성 패턴)
- 폼 (자동 완성, 유효성 검사, 오류 처리)
- 애니메이션 (prefers-reduced-motion, 컴포지터 친화적 변환)
- 타이포그래피 (곡선 따옴표, 줄임표, 표용 숫자)
- 이미지 (크기, 지연 로딩, 대체 텍스트)
- 성능 (가상화, 레이아웃 스래싱, 사전 연결)
- 탐색 및 상태 (URL이 상태 반영, 딥 링크)
- 다크 모드 및 테마 (color-scheme, theme-color 메타)
- 터치 및 상호작용 (touch-action, tap-highlight)
- 로케일 및 국제화 (Intl.DateTimeFormat, Intl.NumberFormat)
