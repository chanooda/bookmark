# PR 리뷰 툴킷

코드 주석, 테스트 커버리지, 오류 처리, 타입 설계, 코드 품질, 코드 단순화를 포괄적으로 검토하는 철저한 풀 리퀘스트 검토를 위한 전문 에이전트 종합 모음입니다.

개요
이 플러그인은 코드 품질의 특정 측면에 각각 집중하는 6개의 전문 검토 에이전트를 묶어 제공합니다. 개별적으로 사용해 특정 부분을 집중 검토하거나, 함께 사용해 포괄적인 PR 분석을 수행할 수 있습니다.

## agents

### 1. comment-analyzer

중점: 코드 주석 정확성과 유지보수성

분석 항목:

- 실제 코드 대비 주석 정확성
- 문서화 완성도
- 주석 노후화 및 기술적 부채
- 오해의 소지가 있거나 구식인 주석

사용 시점:

- 문서화 추가 후
- 주석 변경이 포함된 PR 최종 확정 전
- 기존 주석 검토 시

트리거:

- “주석 정확성 확인”
- “추가한 문서 검토”
- “기술적 부채를 위한 코멘트 분석”

### 2. pr-test-analyzer

초점: 테스트 커버리지 품질 및 완성도

분석 항목:

- 동작 커버리지 vs 라인 커버리지
- 테스트 커버리지의 중요한 공백
- 테스트 품질 및 견고성
- 엣지 케이스 및 오류 조건

사용 시점:

- PR 생성 후
- 새로운 기능 추가 시
- 테스트 철저함 검증 시

트리거:

- "테스트가 충분히 철저한지 확인해줘"
- "이 PR의 테스트 커버리지 검토해줘"
- "중요한 테스트 누락 부분이 있나?"

### 3. silent-failure-hunter

초점: 오류 처리 및 드러나지 않은 위험

분석 항목:

- catch 블록의 무음 실패
- 부적절한 오류 처리
- 부적절한 대체 동작
- 누락된 오류 로깅

사용 시점:

- 오류 처리 구현 후
- try/catch 블록 검토 시
- 오류 처리가 포함된 PR 완료 전

트리거:

- "오류 처리 검토해줘"
- "무음 실패 확인해줘"
- "이 PR의 catch 블록 분석해줘"

### 4. type-design-analyzer

초점: 타입 설계 품질 및 불변성

분석 항목:

- 타입 캡슐화 (1-10점 평가)
- 불변성 표현 (1-10점 평가)
- 타입 유용성 (1-10점 평가)
- 불변성 강제 (1-10점 평가)

사용 시점:

- 새로운 타입 도입 시
- 데이터 모델이 포함된 PR 생성 시
- 타입 설계 리팩토링 시

트리거:

- "UserAccount 타입 설계 검토해줘"
- "이 PR의 타입 설계 분석해줘"
- "이 타입이 강력한 불변성을 가지는지 확인해줘"

### 5. code-reviewer

초점: 프로젝트 가이드라인에 따른 일반 코드 리뷰

분석 항목:

- CLAUDE.md 준수 여부
- 스타일 위반
- 버그 감지
- 코드 품질 문제

사용 시점:

- 코드 작성 또는 수정 후
- 변경사항 커밋 전
- Pull Request 생성 전

트리거:

- "최근 변경사항 검토해줘"
- "모든 게 괜찮은지 확인해줘"
- "커밋 전에 이 코드 검토해줘"

### 6. code-simplifier

초점: 코드 단순화 및 리팩토링

분석 항목:

- 코드 명확성 및 가독성
- 불필요한 복잡성 및 중첩
- 중복 코드 및 추상화
- 프로젝트 표준과의 일관성
- 지나치게 압축되거나 영리한 코드

사용 시점:

- 코드 작성 또는 수정 후
- 코드 리뷰 통과 후
- 코드가 작동하지만 복잡하게 느껴질 때

트리거:

- "이 코드 단순화해줘"
- "더 명확하게 만들어줘"
- "이 구현 개선해줘"

참고: 이 에이전트는 기능을 보존하면서 코드 구조와 유지보수성을 개선합니다.
