# i18n (다국어 지원) 설계

## 배경 및 목표

`mark.` (Chrome 새 탭 확장 프로그램)의 UI 텍스트가 한국어로 하드코딩되어 있다. 한국어, 영어, 중국어(간체), 일본어 4개 언어를 지원하도록 국제화(i18n)를 적용한다.

기존에 `public/_locales/{ko,en,ja,zh_CN}`에 Chrome 확장 프로그램 메타데이터(앱 이름/설명)용 다국어는 이미 구성되어 있으나, 이는 `manifest.json`의 `__MSG_*__` 치환에만 쓰이며 React 앱 내부 UI 텍스트와는 무관하다. 이번 작업은 React 앱 UI 텍스트 전체에 적용하는 것이다.

## 언어 전환 UX

- 기본값: 브라우저 언어(`navigator.language`) 자동 감지
- 수동 전환: 헤더의 설정(gear) 아이콘 버튼 → 설정 모달에서 언어 선택 가능
- 사용자가 수동으로 선택한 언어는 `localStorage`에 저장되어 다음 실행 시에도 유지되며, 저장된 값이 있으면 자동 감지보다 우선한다.

## 라이브러리

`react-i18next` + `i18next-browser-languagedetector` (공식 플러그인)를 사용한다.

- 언어 감지 순서: `localStorage` → `navigator.language` → fallback `en`
- `i18next-browser-languagedetector`가 언어 변경 시 자동으로 `localStorage`에 캐싱하므로, 언어 선택 상태를 위한 별도 zustand 스토어는 만들지 않는다.
- 기존 테마 전환(`src/shared/libs/theme.ts`)이 Context + `localStorage` 패턴을 사용 중이며, i18next도 유사한 영속성 패턴을 라이브러리 차원에서 제공하므로 기존 아키텍처와 결이 맞는다.

## 언어 코드

`ko`, `en`, `ja`, `zh-CN` 4개.

## 파일 구조

```
src/shared/i18n/
  index.ts          # i18next 초기화 (init, languagedetector, react-i18next 플러그인 연결)
  locales/
    en.json
    ko.json
    ja.json
    zh-CN.json
```

- `main.tsx`에서 `src/shared/i18n`을 최상단에서 import하여 앱 렌더링 전에 i18next가 초기화되도록 한다. react-i18next는 별도 Provider 트리 없이 싱글턴 인스턴스를 사용하며, 컴포넌트에서는 `useTranslation()` 훅으로 접근한다.
- 리소스는 단일 네임스페이스(`translation`)로 관리하고, 키는 기능 슬라이스별로 중첩 구조화한다. 예:
  ```json
  {
    "common": { "cancel": "취소", "delete": "삭제", "save": "저장", "add": "추가" },
    "header": { "addBookmark": "북마크 추가" },
    "search": { "placeholder": "...", "noResults": "'{{query}}'에 대한 검색 결과가 없습니다." },
    "bookmark": { ... },
    "folder": { ... },
    "explorer": { ... },
    "itemForm": { ... },
    "deleteConfirm": { ... },
    "settings": { "title": "설정", "language": "언어" }
  }
  ```

## 설정 모달 & 헤더 UI

- **헤더** (`src/widgets/header/ui/header.tsx`): 기존 테마 토글 아이콘 버튼 옆(같은 우측 아이콘 그룹)에 `Settings`(lucide-react) 아이콘 버튼을 추가한다.
- **설정 모달**: 새 feature 슬라이스 `src/features/settings/`
  - `ui/settings-dialog.tsx`: 기존 `DeleteConfirmDialog`/`ItemFormDialog`와 동일하게 `overlay-kit`의 `overlay.open`으로 여는 shadcn `Dialog` 컴포넌트
  - 내용: 제목 "설정"(번역됨), 언어 선택 `Select`(`shared/shadcn/components/ui/select.tsx` 재사용) — 옵션은 각 언어의 자기 언어 이름으로 표기 (한국어 / English / 日本語 / 中文)
  - 언어 선택 즉시 `i18n.changeLanguage(code)` 호출 → 전체 UI가 리렌더링되고 `localStorage`에 자동 저장됨
  - `index.ts`로 export하고 `header.tsx`에서 import하여 사용
  - 지금은 설정 항목이 언어 하나뿐이므로 탭이나 복잡한 레이아웃 없이 단순 리스트형으로 구성한다 (YAGNI). 향후 설정 항목이 늘어나도 리스트에 항목을 추가하는 방식으로 확장 가능하다.

### 제외 범위

- `src/shared/api/bookmark.query.ts`의 목(mock) 데이터 제목("개발", "프론트엔드" 등)은 개발용 샘플 콘텐츠이며 사용자가 실제로 보게 될 UI 문자열이 아니므로 번역 대상에서 제외한다.
- `src/shared/libs/chrome.ts`의 `assertChromeStorage`/`assertChromeBookmarks` 에러 메시지는 확장 프로그램 컨텍스트 밖에서 실행될 때만 발생하는 내부 가드이며 UI에 노출되지 않으므로 제외한다.

## 검증 방법

개발 서버에서 설정 모달을 통해 4개 언어(한국어/영어/일본어/중국어)를 순서대로 전환하며 다음 화면을 직접 확인한다:
- 헤더 (북마크 추가 버튼, 설정 모달)
- 검색 결과 없음 상태
- 북마크/폴더 카드
- 북마크/폴더 생성 폼 다이얼로그, 삭제 확인 다이얼로그
- 탐색기(폴더 상세) 화면

## 범위 밖 (Out of scope)

- Chrome 확장 프로그램 메타데이터(`public/_locales`)는 이미 다국어 처리되어 있으므로 변경하지 않는다.
- 목(mock) 데이터 콘텐츠 번역은 하지 않는다.
- 언어별 날짜/숫자 포맷팅(i18next 외 `date-fns`/`dayjs` locale 연동)은 이번 범위에 포함하지 않는다.
